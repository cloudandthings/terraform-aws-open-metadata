# Claude Code Context

## Repository Overview

Terraform AWS Module Template — a starting point for creating well-structured, tested, and documented Terraform modules for AWS infrastructure. Maintained by [Cloud and Things](https://github.com/cloudandthings).

## Key Conventions

### Terraform and OpenTofu Compatibility

- Module `required_version`: `>= 1.5.0, < 2.0.0`
- Works with Terraform ≥ 1.5.0 and OpenTofu ≥ 1.6.0 (MPL-2.0 open-source fork)
- Primary dev binary: `tofu` (OpenTofu). Terraform also installed for CI validation.
- **Never** use `~> 1.x.y` patch constraints — `~> 1.5.7` resolves to `< 1.6.0`, excluding OpenTofu.
- Stick to the shared feature baseline (TF/OpenTofu 1.6–1.8). Avoid features exclusive to one tool (e.g. Terraform 1.10+ ephemeral resources).

### Module Structure

```text
main.tf         # Module resources (delete null_resource.delete_me placeholder when implementing)
variables.tf    # Input variables — all must have description and type
outputs.tf      # Outputs — all must have description
locals.tf       # Computed local values
terraform.tf    # required_version + required_providers
examples/basic/ # Minimal working example (must stay validate-able)
tests/
  basic.tftest.hcl           # Unit tests — mock providers, no AWS credentials needed
  integration/
    basic.tftest.hcl         # Integration tests — real AWS resources
```

- Resource naming: use `local.naming_prefix` consistently.
- All variables and outputs must have `description`. TFLint enforces this.
- Do not edit README sections between `<!-- BEGIN_TF_DOCS -->` and `<!-- END_TF_DOCS -->` — auto-generated by terraform-docs.

## Running Tests

```sh
# Unit tests (no credentials needed)
tofu test

# Integration tests (requires AWS credentials)
AWS_PROFILE=<profile> tofu test -test-directory=tests/integration
```

## Running Checks

```sh
pre-commit run -a    # All hooks including fmt, validate, tflint, checkov, docs
tofu fmt -recursive
tofu validate
tflint --config=.tflint.hcl
```

## Tools (managed via mise)

All versions are pinned in `mise.toml`. Run `./dev-bootstrap.sh` to install everything.

- **opentofu** — primary Terraform-compatible CLI (`tofu` binary)
- **terraform** — secondary, used for CI version matrix validation
- **tflint** — linting with `tflint-ruleset-aws` plugin (see `.tflint.hcl`)
- **terraform-docs** — auto-generates README docs
- **checkov** — security scanning (pre-commit hook)
- **trivy** — security scanning (CI only, see `trivy-scan.yml`)

## CI/CD

| Workflow | Trigger | Purpose |
|---|---|---|
| `pre-commit-and-tests.yml` | PR | pre-commit + unit tests (ubuntu) + integration tests (CodeBuild/AWS) |
| `terraform-min-max.yml` | PR | Validate with min TF, max TF, and pinned OpenTofu |
| `pr-title.yml` | PR | Enforce conventional commit format |
| `trivy-scan.yml` | PR + push | Security scan |
| `release.yml` | push to main | release-please automated releases |

Dependency updates are managed by **Renovate** (`renovate.json`) — not Dependabot.

## Semantic Commits

`feat:` / `fix:` / `docs:` / `refactor:` / `test:` / `ci:` / `chore:` (chore excluded from changelog)

## Important Notes

- Direct commits to `main`, `master`, `develop` are blocked by pre-commit hook.
- AWS provider version: `>= 5, < 7`
- `null_resource.delete_me` in `main.tf` is a placeholder — delete when building a real module.
- Renovate (not Dependabot) keeps GitHub Actions, Terraform providers, pre-commit hooks, and mise tool versions up to date.
